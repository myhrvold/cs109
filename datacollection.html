Overview

We collected movie scripts from SimplyScripts.com and DailyScript.com. Altogether, we were able to get a total of 396 scripts, as well as their associated IMDB ID numbers from these sites. Next, we used the RottenTomatoes.com API to grab metadata about each movie. We obtained some data that was unavailable on Rotten Tomatoes from IMDB. The code for data collection that we used can be found here

Web Scraping Scripts

We used the BeautifulSoup module in Python to assist with web scraping. For this process, we iterated through each page containing scripts on SimplyScripts.com and DailyScript.com. If a link linked to a script in TXT or HTML format and the website showed a valid IMDB number associated with the script, then our program downloads that script and adds the IMDB number to a list which is used to keep track of the movie scripts collected. If the script is in HTML format, then we convert it to txt format using the html2text.py file available here. Finally, before moving on to the next step, we delete duplicates from the list of IMDB numbers, so that for the next steps we only grab information once for each movie.

Collecting meta data from the Rotten Tomatoes API

We used the Rotten Tomatoes Developer API to grab metadata about each movie, including:

  • Rotten Tomatoes ID
  • Genre (Most movies have multiple genres)
  • Critics' score
  • Audience score
  • Critics rating (Fresh or Rotten)
  • Audience rating (Upright or Spilled)
  • Runtime
  • Release year
  • MPAA Rating
  • Five leading actors and associated character names

All of this data was added to a tab delineated spreadsheet that was used for other analyses. In this phase, we excluded any scripts that were less than 35kb from this spreadsheet since we found that scripts smaller than this often contained garbage data.

Collecting additional data from IMBD

We were unable to find information on movie budgets, boxoffice earnings, and academy award nominations/wins on Rotten Tomatoes. IMDB has a strict policy regarding web scraping, but they do offer a set of large files containing metadata available here. We downloaded the relevant files from there and then manually sorted through them using Excel to get information on the additional fields we needed.
